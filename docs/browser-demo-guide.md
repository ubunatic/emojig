# Emojig Browser Demo Guide

> [!NOTE]
> **Status: Experimental.** The browser demo (Docker/ttyd + WASM modes) is work in progress. Scripts and generated files are untracked. See `issues/09-wasm-build-rootless-mknod.md` for known blockers.

## Overview

`scripts/browser_demo.go` is the unified demo orchestrator that automatically sets up **both Docker (ttyd) and WebAssembly (container2wasm) runtimes** in a single web interface with a toggle switch. It intelligently detects and uses whichever approach is available.

## Quick Start

### Prerequisites

```bash
# Requires:
zig            # Zig compiler
docker         # or podman (for ttyd container)
c2w            # container2wasm (optional for WASM mode)
```

Install container2wasm (optional):
```bash
# Rust/Cargo users
cargo install container2wasm

# Or download pre-built binary from:
# https://github.com/ktock/container2wasm/releases
```

### Run the Demo

```bash
go run scripts/browser_demo.go

# Output:
# ✅ Zig compilation successful
# 📝 Generated Dockerfile and HTML
# 🐳 Building container image...
# ✅ Image ready
# 🔄 Compiling to WebAssembly (1-5 minutes)...
# ✅ Container compiled to WebAssembly
# ✅ Container running on :7681
# ✅ HTTP server running on :8080
#
# 🌐 Web UI → http://localhost:8080/wasm-demo.html
# 🖥️  ttyd   → http://localhost:7681
```

Then open **http://localhost:8080/wasm-demo.html** in your browser.

## Features

### 🐳 Docker Mode (Default)

- **Live streaming terminal** via ttyd WebSocket
- **Perfect interactivity** — all terminal features work (mouse, special keys, etc.)
- **Full fidelity** — runs real shell session with mc, fzf, emojig, etc.
- **Requires**: Docker/Podman container running on port 7681

**Use when**: You want maximum fidelity and interactive testing.

### 🧊 WASM Mode (If Available)

- **Fully self-contained** — no server needed
- **Offline capable** — runs entirely in the browser
- **No latency** — instant local execution
- **Stateless** — each session is isolated; no persistent state across refreshes

**Use when**: You want zero-server deployment or offline capability.

**Available only if**: `c2w` was installed and compilation succeeded.

## Toggle Behavior

Click the buttons at the top of the terminal:

| Mode | Terminal | Server | Performance | Use Case |
|------|----------|--------|-------------|----------|
| **Docker (Live)** | ✅ Full xterm.js | Streaming from ttyd | Network dependent | Development, full testing |
| **WASM (Offline)** | ✅ xterm.js in iframe | None (static) | Instant local | Distribution, demos |

### Switching Modes

1. Click **🐳 Docker (Live)** — connect to live ttyd WebSocket on port 7681
2. Click **🧊 WASM (Offline)** — load embedded WASM container from `/wasm-out/index.html` (lazy-loaded on first click)

Both modes use the same xterm.js terminal emulator for consistent UX.

## What Gets Generated

```
scripts/
├── browser_demo.go        ← The unified orchestrator script
├── wasm-demo.html         ← Generated: web UI with Docker/WASM toggle
├── demo.Dockerfile        ← Generated: container definition
│
└── wasm-out/              ← Generated by c2w (if available)
    ├── index.html         ← container2wasm's terminal UI
    ├── out.wasm           ← Compiled container (10-20 MB)
    └── out.js             ← WASM loader and runtime
```

## Troubleshooting

### "c2w not found" or "compilation failed"

**Symptom**: WASM tab shows "⚠️ WASM Build Not Available"

**Solution**: 
1. Install container2wasm: `cargo install container2wasm` (requires Rust)
2. Verify it's in PATH: `which c2w`
3. Re-run `go run scripts/browser_demo.go`

Docker mode will still work regardless.

### Docker mode not connecting

**Symptom**: Docker tab shows connection error

**Solution**:
1. Ensure Docker/Podman is running: `docker ps`
2. Check ttyd port 7681 is accessible: `curl -i http://localhost:7681/`
3. Restart: Kill the script with Ctrl+C and run again

### WASM iframe not loading

**Symptom**: WASM tab is blank or shows 404

**Solution**:
1. Check that HTTP server is running on port 8080: `curl http://localhost:8080/`
2. Verify `/wasm-out/` exists: `ls -la scripts/wasm-out/`
3. Check browser console (F12 > Console) for errors

### Emoji misalignment in WASM mode

**Note**: WASM mode runs in a browser environment with different font rendering. Emoji width calculation may differ from Docker mode. This is expected and documented in [[browser-runtime-options.md]].

## Unified Script

The demo script automatically:
- ✅ Always starts **Docker mode** (ttyd) for live streaming
- ✅ Optionally compiles to **WASM mode** if `c2w` is available
- ✅ Provides a toggle interface to switch between modes
- ✅ Gracefully falls back to Docker-only if c2w is not installed

**No configuration needed** — just run and the script handles everything!

## Architecture

```
Browser UI (wasm-demo.html)
    ↓
    ├─→ Docker Mode
    │   └─→ WebSocket to ttyd (port 7681)
    │       └─→ Container shell (emojig, mc, fzf, etc.)
    │
    └─→ WASM Mode
        └─→ <iframe src="/wasm-out/index.html">
            └─→ container2wasm WASM module
                └─→ Linux (Alpine) + bash + emojig + mc + fzf
```

## Performance Comparison

### Docker Mode
- **Startup**: <100ms (WebSocket handshake)
- **Latency**: Network-dependent (typically 50-200ms)
- **Memory**: Browser + xterm.js (~5-10 MB)
- **Network**: Continuous streaming

### WASM Mode
- **Startup**: 2-5 seconds (WASM module load + initialization)
- **Latency**: Instant local execution
- **Memory**: Browser + xterm.js + WASM container (10-30 MB)
- **Network**: One-time 10-20 MB download
